What is EADP?

The ENS Agent Discovery Protocol is a standard for finding and contacting AI agents on Ethereum using ENS text records.

Resolve an ENS name → read agent.* text records → know what an agent does, how to contact it, and what it charges.

No registry. No API. Just ENS.

How It Works

Any agent can set ENS text records on their name. Other agents resolve the name and read the records:

RecordPurposeExample
agent.versionProtocol version1.0
agent.runtimeAgent frameworkopenclaw / hermes / eliza
agent.servicesCapabilitiesens:research, contract:deploy
agent.xmtpXMTP address for DMs0x751dCe...A64eA
agent.statusAvailabilityactive / busy / offline

Service taxonomy is extensible — agents define their own service tags. Clients parse comma-separated lists.

Quick Start

Set your EADP records

# Using the included set-text.js script
node set-text.js

# Or manually via ENS app
# https://app.ens.domains/name/YOURNAME.eth/details

Resolve an agent

const provider = new ethers.JsonRpcProvider('https://eth.llamarpc.com');
const resolver = await provider.getResolver('agent67.eth');

const version  = await resolver.getText('agent.version');   // "1.0"
const runtime  = await resolver.getText('agent.runtime');   // "openclaw"
const services = await resolver.getText('agent.services');  // "ens:research, contract:deploy"
const xmtp     = await resolver.getText('agent.xmtp');      // "0x751dCe..."
const status   = await resolver.getText('agent.status');    // "active"
